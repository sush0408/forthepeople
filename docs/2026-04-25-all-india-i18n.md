# All-India Internationalisation Support

## Decision

ForThePeople now supports URL routing and language selection for English plus the 22 Eighth Schedule languages. Kannada remains a completed pilot translation. Other Indian language routes are enabled as preview routes and intentionally fall back to English copy until reviewed translations are added.

## Locale Coverage

Supported route prefixes: `en`, `kn`, `hi`, `te`, `ta`, `ml`, `mr`, `bn`, `gu`, `pa`, `or`, `as`, `ur`, `sa`, `ne`, `sd`, `ks`, `doi`, `kok`, `mni`, `brx`, `sat`, `mai`.

The shared registry lives in `src/i18n/locales.ts` so middleware, page code, and UI selectors use the same source of truth.

## Data Flow

1. `src/i18n/routing.ts` passes the full locale list to `next-intl`.
2. `src/proxy.ts` matches every supported locale prefix.
3. `src/components/layout/Header.tsx` shows every supported language as selectable.
4. `src/dictionaries/index.ts` returns the Kannada dictionary for `kn` and English for all untranslated preview locales.

## Next Translation Sprint

Add reviewed dictionaries one language at a time, starting with Hindi, Tamil, Telugu, Marathi, Bengali, Malayalam, Gujarati, Punjabi, Odia, Assamese, and Urdu. Each added dictionary should be wired into `src/dictionaries/index.ts` and then flipped from `fallback` to `complete` in `src/i18n/locales.ts`.
